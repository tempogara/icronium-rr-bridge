Manuale Tecnico

Riferimento tecnico per setup, runtime, persistenza, endpoint e struttura dell'applicazione.

Apri manuale utente HTML · Apri README.md

Stack

  • Java 21
  • Spring Boot 3.3.5
  • Gradle Wrapper
  • HTML + Bootstrap + STOMP/WebSocket
  • Persistenza JSON su disco

Avvio

Dalla root del progetto:

./gradlew bootRun

URL: http://localhost:8087

Configurazione runtime

  • Porta HTTP: 8087
  • Timeout sessione: 12h
  • Log file: logs/connector-rr.log

File: src/main/resources/application.properties

Pagine

  • /index.html
  • /dashboard.html
  • /gara-dettaglio.html?eventId=<RRID>
  • /timingpointView.html?eventId=<RRID>
  • /rr-live.html?eventId=<RRID>

Sessione e WebSocket

  • Auth REST: /api/auth/*
  • Keepalive: GET /api/auth/keepalive
  • Handshake WS: /ws
  • Topic: /topic/messages, /topic/gare-updates, /topic/rr-live-messages, /topic/rr-live-updates

La sessione RR è gestita lato backend. Il frontend non conserva il pw RR.

Persistenza

  • data/gara-dettaglio/<eventId>.json
  • data/rr-live/<eventId>.json

I dati persistiti includono configurazione righe, stati, contatori, blacklist e storici recenti.

Scheduler

Race Detail

  • partenza su PLAY
  • primo download immediato
  • polling regolato da Download every
  • lettura incrementale file
  • righe con * scartate in ingresso
  • post batch massimo 300
  • in caso di errore post il contatore non avanza

RR Live

  • sync API RR Live per evento
  • polling per riga
  • RESET azzera la memoria dell'ultimo file inviato

API principali

  • POST /api/auth/login
  • POST /api/auth/login-local
  • GET /api/gare
  • POST /api/gare/{eventId}/sync
  • GET /api/gare/{eventId}/rows
  • POST /api/gare/{eventId}/rows/{rowId}/actions/{action}
  • POST /api/rr-live/sync
  • GET /api/rr-live/state/{eventId}

Il dettaglio completo è nel README tecnico.

Log e troubleshooting

  • log applicativo: logs/connector-rr.log
  • porta applicazione: 8087
  • se il browser mostra errori websocket, verifica /ws e la porta attiva
  • se la sessione scade, verifica il keepalive e lo stato auth
