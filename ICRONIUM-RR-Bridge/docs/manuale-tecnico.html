Manuale Tecnico

Riferimento tecnico per setup, runtime, persistenza, endpoint e struttura dell'applicazione.

Apri manuale utente HTML · Apri README tecnico

Stack

  • Java 21
  • Spring Boot 3.3.5
  • Gradle Wrapper
  • HTML + Bootstrap + WebSocket
  • Persistenza JSON su disco

Modalità

  • RR
  • RR_LOCALE
  • TZERO

In TZERO la root gara è persistita in data/tzero/config.json.

Pagine

  • /dashboard.html
  • /gara-dettaglio.html (Settings)
  • /timingpointView.html
  • /fileView.html
  • /readerView.html
  • /rr-live.html
  • /speaker-tp.html?token=...

Persistenza

  • data/gara-dettaglio/<eventId>.json
  • data/rr-live/<eventId>.json
  • data/reader-view/<eventId>.json
  • data/speaker-tokens.json
  • data/tzero/config.json

Per le gare TZero esistono anche:

  • <gara>/config/timingpoints.json
  • <gara>/config/sources.json

Scheduler

  • Race Detail: polling file e post RR
  • TZero: post simulato per calcolo discarded
  • Reader View: scan periodico file odierni per reader
  • Mirror TZero: le sorgenti remote vengono copiate anche in download/

Reader View

  • reader types: ICRON, UBIDIUM, FEIBOT
  • location: LAN, WAN, FS
  • Scan LAN: discovery su 192.168.x.* via /files
  • Scan WAN: discovery manuale reader attivi via endpoint web

Speaker

La pagina speaker usa un token persistito lato backend. Il frontend genera QR ma il link pubblico viene prodotto dal backend con IP LAN 192.168.* quando disponibile.
