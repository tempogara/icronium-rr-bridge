Accesso

Apri il browser su http://localhost:8087.

La home presenta due modalità: Login to RR e Login to RR local.

Login

Login to RR

Usa questa modalità quando lavori con Race Result remoto.

  • Inserisci userid
  • Inserisci password
  • Premi Login to RR

Le credenziali vengono ricordate nel browser e riproposte agli accessi successivi.

Login to RR local

Usa questa modalità quando lavori contro Race Result locale. Non servono credenziali RR.

Dashboard

Dopo il login si apre la dashboard con l'elenco delle gare.

  • ID
  • Event name
  • Event date

Le gare sono ordinate per data decrescente. Per entrare in una gara premi Select.

In basso trovi sempre la sezione Messages, aggiornata via WebSocket.

Race Detail

È la vista principale per gestire i file di passaggi.

Nella toolbar trovi Dashboard, Open RR, RR Live, TP View, RR ID, nome evento e data evento.

Sync iniziale

Quando entri nella pagina viene eseguito automaticamente un SYNC.

Il sync carica da RR:

  • Timing Point
  • tabella BIB-CHIP
  • dati anagrafici atleta

Puoi rilanciarlo manualmente con il bottone SYNC.

Gestione Source

Un source è un file di passaggi da monitorare. Può essere:

  • file locale
  • URL di rete locale 192.168...
  • URL remoto HTTP/HTTPS

Per aggiungere una sorgente usa Add file passings e compila:

  • Source
  • Timing point
  • Download every
  • Sync

Azioni riga

  • Play
  • Stop
  • Rewind
  • Edit
  • Duplicate
  • Delete

Stati

  • STOPPED
  • RUNNING
  • ERROR

Se un download fallisce la riga va in ERROR, ma continua a riprovare finché non la fermi manualmente.

Totals e Last 10 Cycles

Totals

Ogni riga mostra 4 celle:

  • READ: righe lette
  • SENT: righe inviate
  • DISC: righe scartate
  • UNIQUE: chip unici letti

Le celle sono cliccabili per aprire i dettagli.

Last 10 Cycles

Mostra gli ultimi 10 cicli della riga:

  • DL: quante righe sono state lette nel ciclo
  • OUT: quante righe sono state inviate nel ciclo

Cliccando una cella apri il dettaglio del singolo ciclo.

Popup operative

File content

  • sottotitolo con URL/path completo
  • ordine iniziale dal più recente al più vecchio
  • bottone Newest/Oldest
  • campo filtro
  • colonna atleta
  • bottone Black list

Unique

Mostra i chip unici letti con ordinamento e filtro.

Discarded

Mostra le righe non riconosciute rispetto a BIB-CHIP. Anche qui puoi usare la blacklist.

Blacklist

La blacklist è per-riga. Un codice escluso su un source non viene escluso automaticamente sugli altri.

Se blacklisti un codice:

  • sparisce dalle popup
  • non viene più conteggiato
  • non viene più inviato

La popup Blacklist in toolbar permette di ripristinare i codici con Restore.

TP View

TP View raggruppa i source per timing point.

Per ogni timing point vedi:

  • nome timing point
  • lista dei source con stato
  • box Totals aggregato

Le popup aggregate di READ e UNIQUE mostrano anche il file di provenienza.

RR Live

Questa pagina gestisce il flusso basato sulle API live di Race Result.

  • sync iniziale automatica
  • azioni PLAY, STOP, RESET, DELETE
  • global actions
  • area Messages

RESET azzera la memoria dell'ultimo file spedito, così il prossimo PLAY riparte da zero.

Messaggi e Log

La sezione Messages serve per monitorare download, upload, errori e attività scheduler. È richiudibile e ricorda lo stato nel browser.

I log completi sono scritti in logs/connector-rr.log.

Suggerimenti operativi

Avvio gara

  1. Fai login
  2. Seleziona la gara in dashboard
  3. Verifica SYNC
  4. Controlla BIB-CHIP e Timing Points
  5. Aggiungi i source necessari
  6. Premi PLAY
  7. Monitora Totals, Last 10 cycles e Messages

Se un file va in errore

  1. Controlla path o URL
  2. Verifica lo stato ERROR
  3. Guarda i messaggi in basso
  4. Usa STOP solo se vuoi fermare i tentativi automatici
